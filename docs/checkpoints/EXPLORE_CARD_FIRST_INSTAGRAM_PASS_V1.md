# EXPLORE CARD-FIRST INSTAGRAM PASS V1

## Purpose
Refactor Explore so it behaves like a card-first collector browsing surface instead of a catalog utility page.

## Current Problems
- the screen still reads like catalog/discovery infrastructure
- cards are visually secondary to structure and controls
- the mid-page `Sets` block interrupts momentum
- the surface lacks the emotional pull of a collector feed
- the app now flows better, but the Explore presentation still feels too institutional

## Existing Strengths To Preserve
- search bar
- browse sets entry point
- filters if still useful
- grid/list control if still useful
- direct add-to-vault affordance
- tap-to-action-hub behavior
