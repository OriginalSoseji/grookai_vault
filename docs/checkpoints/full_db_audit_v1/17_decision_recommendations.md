# DECISION RECOMMENDATIONS

1. Decide whether canonical printed identity must become string-first. The live provenance surface already contains alphanumeric printed-number carriers, so a numeric-only model is not supported by current evidence.
2. Decide whether a provenance-normalization contract is now required before parent repair. Stored truth exists in linked raw payload number fields and provenance carriers, but it is not uniformly normalized into parent identity fields.
3. Decide whether remote identity rollout must remain blocked until the supported null-parent surface is repaired or explicitly reclassified. `10620` supported rows still lack both parent `set_code` and `number`.
4. Decide whether legacy parent uniqueness on remote `card_prints` must be reconciled before any future BA or identity-subsystem execution resumes remotely.
5. Decide whether symbolic or promo-style printed identities need a dedicated rule set instead of being treated as edge-case numeric variants.
